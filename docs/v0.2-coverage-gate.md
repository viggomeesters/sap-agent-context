# SAP Agent Context v0.2 coverage gate

`schema/completeness-matrix.yaml` remains the starter safety gate used by
`make check`: it proves the repository can build and serve representative starter
bundles without schema, freshness, or evaluation regressions.

`schema/sap-agent-context-v0.2-coverage.yaml` is stricter. It is the fill-campaign
readiness gate for turning this repository into a genuinely useful public clone
for AI agents.

## Why this gate is expected to fail today

The current repository has 64 valid items and the starter queries pass, but some
coverage is still seed-level:

- material/product and business-partner context has only a few concrete field
  definitions and field maps;
- migration cockpit/template-label knowledge is still an outline rather than a
  rich table/field/value-source graph;
- domain packs mostly prove that a representative bundle can be produced, not
  that the domain is deep enough for repeated FO generation;
- source specificity and release applicability are present, but not at the level
  needed for a colleague to trust broad autonomous agent use.

That failure is intentional. A green starter gate plus a red v0.2 gate is the
honest state between merge-closure and full-fill completion.

## How to run

```bash
uv run sap-agent-context audit-completeness \
  --matrix schema/sap-agent-context-v0.2-coverage.yaml
```

Expected current result: `status: failed` with critical total-count and important coverage/domain/source
findings. Later fill tasks should reduce this report until v0.2 passes.

## What v0.2 checks beyond starter coverage

- higher total item and per-kind minimums;
- concrete SAP field definitions and field maps per domain;
- exact-page/catalog-entry source specificity, not only SAP root links;
- release applicability metadata at scale;
- domain-specific kind counts for master data, migration cockpit, AP,
  procurement, sales/output, authorizations, integrations, and
  extensibility/analytics;
- representative queries that must contain the right item kinds and quality
  dimensions instead of passing via adjacent generic context.

## Non-negotiables

- Do not weaken v0.2 just to make the campaign green.
- Do not copy proprietary SAP documentation; use source URLs and own concise
  claims.
- Do not add customer/client/project data, screenshots, tenant URLs, secrets, or
  internal mappings.
- Keep `freshness.review_after` and `freshness.expires_at` on every knowledge
  item.
- Keep `bundle_kind: sap_fo_context_bundle` until an explicit migration plan
  exists.
